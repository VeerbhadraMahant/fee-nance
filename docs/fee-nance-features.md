# Fee-Nance Features

This document lists the major features currently implemented in Fee-Nance.

## 1. Authentication and User Access

- User registration with email and password.
- User login with credentials (NextAuth).
- Optional Google OAuth login when credentials are configured.
- Session-based authentication for protected routes.
- Route protection for private app areas such as dashboard, finance, and groups.
- Private API protection under `/api/private/*`.

## 2. Profile and Preferences

- Fetch current user profile and preference data.
- Update profile fields (such as display name and preferences).
- Per-user settings persisted in the database.

## 3. Personal Finance Features

### 3.1 Transaction Management

- Create income and expense transactions.
- Edit existing transactions.
- Delete transactions.
- List transactions with filtering by date range.
- Filter transactions by type (`income` or `expense`).
- Sort transactions by fields such as date, amount, title, or creation time.
- Paginate transaction lists.

### 3.2 Recurring Transactions

- Support for recurring transactions (monthly and yearly cadence).
- Endpoint to run recurring generation and create due entries.

### 3.3 Category Management

- Built-in system categories.
- User-created custom categories.
- Create, update, and delete custom categories.
- Category-aware finance summaries.

### 3.4 Budget Management

- Create budgets for planned spending.
- List budgets with date filtering.
- Update budget details.
- Delete budgets.
- Budget cycles supported: monthly, quarterly, yearly.

### 3.5 Finance Aggregation and Insights

- Unified finance aggregate payload (categories, transactions, budgets).
- Income vs expense summary.
- Running balance and total calculations.
- Category-wise spending summaries.
- Date-range-based analytics.

## 4. Dashboard Features

- Protected dashboard experience for authenticated users.
- Summary cards for financial overview.
- Category breakdown for spending visibility.
- Monthly trend data for reporting.
- Group count and quick snapshot information.
- Date-range filtering support through API query parameters.

## 5. Group Expense Management

### 5.1 Group Creation and Membership

- Create groups.
- Auto-create owner membership for newly created groups.
- Join groups via invite code.
- List user groups with pagination, sorting, and search.
- Fetch group detail view.

### 5.2 Shared Expense Recording

- Create group expenses with multiple payers.
- List group expenses with filters and sorting.
- Filter by date range and creator.
- Pagination support for expense history.

### 5.3 Split Logic

- Supported split modes:
  - Equal split
  - Custom amount split
  - Percentage split
- Validation that payer/split totals match the total expense amount.
- Business rule enforcement to prevent inconsistent group expense state.

### 5.4 Balances and Settlements

- Compute per-member net balances.
- Generate simplified pairwise settlement suggestions.
- Record manual settlements.
- List settlement history with date/user filters, sorting, and pagination.
- Idempotency support for settlement creation (`x-idempotency-key` or `idempotency-key`).

## 6. Data Integrity, Validation, and Error Handling

- Request validation using schema-driven checks.
- Input parsing and business-rule enforcement in private APIs.
- Structured error semantics:
  - `401` unauthorized
  - `403` forbidden
  - `404` not found
  - `422` validation or business-rule failure
  - `500` unexpected server error
- Logger and error-boundary support at application level.

## 7. Security and Authorization Model

- Session-required access for private APIs.
- Ownership and membership checks for protected resources.
- Auth guard flow before private data operations.
- Permission checks in group and category operations.

## 8. API Surface (Implemented Modules)

- Authentication: registration and NextAuth handlers.
- Profile module: current user read/update.
- Dashboard module: summary and trend endpoints.
- Finance module: aggregate, transactions, categories, budgets.
- Group module: groups, membership, expenses, balances, settlements.

## 9. Runtime and Platform Features

- Runtime database: MongoDB Atlas via Mongoose.
- API-first App Router architecture in Next.js.
- Type-safe codebase with TypeScript.
- Validation layer with Zod.
- Demo data seeding for repeatable testing and demo flow.

## 10. DBMS Academic Features and Deliverables

- SQL script bundle for relational equivalents:
  - DDL and schema definitions
  - DML demo inserts
  - JOIN and subquery examples
  - GROUP BY and HAVING examples
  - Trigger/procedure/function/cursor equivalence notes
- Query mapping document from SQL concepts to MongoDB runtime implementation.
- Reproducible reporting output generated by project script (`npm run dbms:report`).

## 11. Demo-Ready Capabilities

- Seeded demo accounts with sample data.
- End-to-end personal finance and group expense showcase.
- Reproducible report generation for viva/submission evidence.
- Documentation set for architecture, API references, query mapping, and QA checks.

## 12. Feature Summary

Fee-Nance combines two normally separate product areas into one app:

- Personal money management (transactions, categories, budgets, summaries).
- Collaborative expense management (groups, splitting, balances, settlements).

Together, these features provide practical daily utility while also demonstrating solid DBMS concepts through mapped relational deliverables.
